## GET /v1/sessions/{session_id}/messages

List all messages in a session, including user messages, agent responses, and tool results.

### Path Parameters

<ParamField path="session_id" type="string" required>
  The unique identifier of the session
</ParamField>

### Response

Returns a non-paginated list of all messages in the session.

<ResponseField name="data" type="array">
  Array of message objects
  
  <ResponseField name="data[].id" type="string">
    Message identifier (UUID v7)
  </ResponseField>
  
  <ResponseField name="data[].role" type="string">
    Message role: `user`, `agent`, or `tool`
  </ResponseField>
  
  <ResponseField name="data[].content" type="array">
    Array of content parts. Can include text, tool calls, and tool results.
  </ResponseField>
  
  <ResponseField name="data[].created_at" type="string">
    ISO 8601 timestamp
  </ResponseField>
  
  <ResponseField name="data[].controls" type="object">
    Optional control parameters (present on user messages if provided)
  </ResponseField>
  
  <ResponseField name="data[].metadata" type="object">
    Optional message metadata
  </ResponseField>
</ResponseField>

<ResponseField name="total" type="integer">
  Total number of messages
</ResponseField>

### Example Request

```bash
curl https://api.everruns.com/v1/sessions/{session_id}/messages \
  -H "Authorization: Bearer YOUR_API_KEY"
```

## Message Reconstruction

Messages are reconstructed from events. The following event types contribute to the message list:

| Event Type | Role | Content Source |
|------------|------|----------------|
| `input.message` | `user` | `data.message.content` |
| `output.message.completed` | `agent` | `data.message.content` (may include tool calls) |
| `tool.completed` | `tool` | `data.result` (tool execution results) |

## Content Types

### Text Content

```json
{ "type": "text", "text": "Message text" }
```

### Image File Content

```json
{
  "type": "image_file",
  "image_id": "01933b5a-...",
  "filename": "photo.png"
}
```

### Tool Call Content

```json
{
  "type": "tool_call",
  "id": "call_123",
  "name": "get_weather",
  "arguments": { "city": "Tokyo" }
}
```

### Tool Result Content

```json
{
  "type": "tool_result",
  "tool_call_id": "call_123",
  "content": [
    { "type": "text", "text": "Result data" }
  ]
}
```

## Error Responses

<ResponseField name="error" type="string">
  Error message
</ResponseField>

<ResponseField name="status" type="integer">
  HTTP status code
</ResponseField>

### Common Errors

- **404 Not Found**: Session does not exist
- **500 Internal Server Error**: Server error during retrieval
